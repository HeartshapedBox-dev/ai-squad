# Constraints

## 절대 금지

- 운영 환경변수명 임의 변경 금지
- 인증/권한 로직 임의 제거 금지
- DB 컬럼 삭제 금지
- 기존 API 응답 구조 임의 변경 금지
- 민감정보 로그 출력 금지
- 관련 없는 리팩토링 금지
- 패키지 매니저 임의 변경 금지
- Docker/배포 설정 임의 변경 금지
- 외부 API 요청/응답 구조 임의 단정 금지
- 기존 주석 삭제 금지

## 주의

- 외부 API 연동부는 timeout/error handling을 고려한다.
- 배포 영향이 있으면 env/migration 여부를 명시한다.
- 기존 테스트가 있으면 깨지지 않게 유지한다.
- 운영 데이터에 영향이 있으면 먼저 위험 요소를 설명한다.
- 대량 데이터 조회는 pagination/index 필요 여부를 검토한다.
- 로그에는 개인정보, 토큰, API Key, 비밀번호를 남기지 않는다.

## 수정 전 확인할 것

- 어떤 파일을 수정할지
- 왜 수정해야 하는지
- 기존 기능에 영향이 있는지
- DB 변경이 있는지
- 환경변수 변경이 있는지
- 배포 작업이 필요한지
- 테스트 방법은 무엇인지
