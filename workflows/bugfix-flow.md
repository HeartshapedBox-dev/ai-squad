# Bugfix Flow

## 목적

버그를 추측으로 고치지 않고, 원인 분석 → 최소 수정 → 회귀 테스트 순서로 처리한다.

## 사용 상황

- API 에러
- 배포 실패
- DB 조회 오류
- 인증/권한 오류
- 프론트 연동 오류
- 운영 로그 기반 장애

## 기본 흐름

Bug Analysis → Backend/Frontend/Infra/Database → Codex → Reviewer → Tester

## 축소 흐름

백엔드 버그:

Backend → Codex → Reviewer → Tester

배포/서버 버그:

Infra → Codex → Reviewer

DB 버그:

Database → Backend → Codex → Reviewer

## 진행 순서

1. bug-fix.md 템플릿으로 문제를 정리한다.
2. 가능한 원인을 3개 이내로 좁힌다.
3. 담당 에이전트를 선택한다.
4. 최소 수정안을 만든다.
5. Codex로 수정한다.
6. reviewer.md로 수정 범위를 리뷰한다.
7. tester.md로 재현 테스트와 회귀 테스트를 만든다.

bug-fix.md 기준으로 아래 버그의 원인을 분석해줘.

문제:
[문제 현상]

로그:
[에러 로그]

최근 변경:
[최근 수정 내용]

먼저 가능한 원인을 3개 이내로 좁히고,
각 원인을 확인하는 방법을 알려줘.
아직 수정안은 확정하지 마.

- 인증/권한 로직 임의 변경 금지
- 수정 후 재현 테스트 방법 작성

## 완료 기준

- 원인이 설명됨
- 최소 범위로 수정됨
- 기존 기능 영향이 검토됨
- 재현 테스트가 통과함
- 회귀 테스트 범위가 정리됨
